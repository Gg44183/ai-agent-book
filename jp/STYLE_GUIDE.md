---

## 基本原則

**日本語読者向けの技術書として、日本語表記を基本とし、初出時に英語を併記する。**

---

## 用語統一ルール

### 1. 「エージェント」の表記

| 場面 | 表記 | 例 |
|------|------|-----|
| 章タイトル・見出し | エージェント | 「第1章 エージェントの本質」 |
| 正文・初出 | エージェント（Agent） | 「エージェント（Agent）とは自律的に...」 |
| 正文・2回目以降 | エージェント | 「エージェントがツールを呼び出す」 |
| 複合技術用語 | 英語保持 | Multi-Agent, ReAct Agent, Tool Agent |
| コード・変数名 | 英語 | `AgentCore`, `agent_id`, `SimpleTaskAgent` |
| 固有名詞・製品名 | 原文保持 | Shannon Agent, LangGraph Agent |

### 2. 保持すべき英語表記（変更不可）

以下は技術用語として英語のまま保持：

```
- Multi-Agent（マルチエージェントの文脈では「マルチエージェント」も可）
- ReAct Agent
- Tool Agent
- Planning Agent
- Background Agent
- Supervisor Agent
- Worker Agent
- LLM Agent
- AI Agent（タイトルや固有表現）
```

### 3. コードブロック内

コードブロック（```で囲まれた部分）内は一切変更しない。
